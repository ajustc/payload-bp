# Project Overview

Payload 3.0 blank template built with Next.js 16, React 19, PostgreSQL, and TypeScript. Headless CMS with admin UI and GraphQL/REST APIs.

## Tech Stack

- **CMS**: Payload 3.83.0
- **Framework**: Next.js 16.2.3 (App Router)
- **UI**: React 19.2.4, @payloadcms/ui 3.83.0
- **Database**: PostgreSQL via @payloadcms/db-postgres
- **Editor**: Lexical rich text editor
- **Image Processing**: Sharp 0.34.2
- **Testing**: Vitest (integration), Playwright (E2E)
- **Package Manager**: pnpm

## Key Configuration Files

- `src/payload.config.ts` - Main config: collections, database, secret, plugins
- `next.config.ts` - Next.js config with image patterns, webpack extensions, turbopack
- `tsconfig.json` - TypeScript config with path aliases (@/*, @payload-config)
- `.env` - DATABASE_URL, PAYLOAD_SECRET (use .env.example as template)

## Security Notes

- **PAYLOAD_SECRET**: Required for session encryption and CSRF protection. Generate cryptographically random string.
- **Database**: Uses connection pooling via pg. Ensure DATABASE_URL uses SSL in production (`?sslmode=require`).
- **Auth**: Users collection has `auth: true` - provides login, session management, access control.
- **Media**: Public read access by default (`read: () => true`). Restrict in production if needed.
- **CORS**: Configure in payload.config.ts for API access control.
- **Rate Limiting**: Add plugin/ middleware for production API protection.

## Architecture Patterns

- **Route Groups**: `(frontend)` for public, `(payload)` for admin/API separation
- **Server Actions**: Payload uses `handleServerFunctions` for admin server actions
- **Path Aliases**: `@/*` maps to `src/*`, `@payload-config` points to `payload.config.ts`
- **Auto-generated Files**: Admin layout, API routes are regenerated by Payload - do not modify
- **Type Generation**: Run `generate:types` after schema changes to update TypeScript definitions

## Collections

- **Users**: Built-in auth collection with email field. Extend with additional fields as needed.
- **Media**: Upload collection with required `alt` text for accessibility.

## Adding New Collections

1. Create file in `src/collections/YourCollection.ts`
2. Import and add to `collections` array in `payload.config.ts`
3. Run `pnpm generate:types` to update types
4. Restart dev server

## Frontend Data Fetching

```typescript
import { getPayload } from 'payload'
import config from '@/payload.config'

const payload = await getPayload({ config: await config })
const docs = await payload.find({ collection: 'your-collection' })
```
